# My Sweetheart

A single-page lyric experience for a genuinely public-domain 1910 love song
— **Vintage Sunset**: a warm brick-red -> orange -> gold gradient, bold
rounded kinetic typography paired with a brush-script accent, real
photographic vintage objects (a gramophone, a rotary phone, a pocket watch…)
drifting through Three.js parallax depth, and thin line-art doodles. Every
bit of motion — line staging, camera, background mood, photo-layer drift —
is a real [Theatre.js](https://www.theatrejs.com/) keyframe sequence, not
hand-rolled per-frame math. There is exactly one control: a play button.

## What's actually in this build

- **Real audio, real timing, down to the word.**
  `public/audio/let-me-call-you-sweetheart.mp3` is a 1924 Jubileers
  vocal-quartet recording of the fully public-domain 1910 song "Let Me Call
  You Sweetheart" (Leo Friedman / Beth Slater Whitson). Every lyric and
  every word in `src/lyrics-data.mjs`'s `LYRICS` array carries its own
  start/end timestamp, produced by running
  [faster-whisper](https://github.com/SYSTRAN/faster-whisper) with
  word-level timestamps directly on that recording. See `ATTRIBUTION.md`
  for the full method.
- **Every line, the camera, the background mood, and every parallax layer
  are real Theatre.js sequences.** `@theatre/core` drives
  position/opacity/scale/rotation/blur/font-weight for every lyric line, a
  slow breathing camera dolly, a hue/warmth crossfade between song
  sections, and the drift/scale/opacity of every photo and doodle layer —
  all from `public/assets/theatre-state.json`, an on-disk Theatre.js
  project state (the same format Theatre.js Studio itself exports).
  `src/main.js` does nothing more than scrub `sheet.sequence.position` to
  `audio.currentTime` every frame and apply whatever each object's
  `onValuesChange` hands back. See `tools/build-theatre-state.mjs` for why
  that file is generated by a script rather than authored by hand in
  Theatre.js Studio's visual editor.
- **A continuous stream of real photographic cutouts, not three isolated
  hero moments.** `PHOTO_LAYERS` in `src/lyrics-data.mjs` lists ~17 moments
  across the whole song — a rotary phone for "let me call you," a dove into
  the dreaming verse, a pocket watch at the finale — each a licensed Adobe
  Stock photo with its background removed (see `ATTRIBUTION.md`), rendered
  as a camera-facing Three.js sprite. Each is locked to a depth tier
  (`DEPTH_TIERS`) that controls how big, how fast, and — critically — which
  lateral *lane* it drifts in, so a large fast-moving cutout can never end
  up sitting on top of the lyric text; see `src/scene/photoLayers.js`.
- **Thin line-art doodles, not flat colored stickers.** The
  hearts/sparkles/star/x accents drifting near the photo layers are
  [Lucide](https://lucide.dev) icons (MIT), matching the reference
  design's clean outline look instead of a flat-icon clip-art look.
- **Two typefaces, one deliberate accent.** Baloo 2 (bold, rounded) carries
  almost all the lyric type; Caveat (loose brush script) is reserved
  entirely for the word "sweetheart" and the finale's "with love" — one
  second typeface used on purpose, not a whole second type system.
- **Word-by-word kinetic entrance + sung/unsung highlighting**, both driven
  directly off each word's real timestamp: a 3D rotate-and-blur-in reveal
  when a word's about to be sung, then a progress-wipe underline exactly
  when it is. Both stay outside Theatre.js on purpose — discrete
  timestamp-triggered state, not a continuously tweened value.
- **A real ending, not a silent reset.** When the song actually finishes, a
  real rose photo pulses like a heartbeat, a few more photo cutouts drift
  in around the message, and "From Naveen, with love" fades up.
- **Mobile-aware throughout**: capped device-pixel-ratio and reduced
  particle/sprite complexity on small screens, a responsive font size, safe
  area insets, and a `prefers-reduced-motion` path that freezes the
  decorative motion in both the DOM and the WebGL layer.

## Repo layout

```
index.html                     Vite entry (HTML shell only — see src/main.js)
src/
  main.js                       boots audio, Theatre.js, Three.js, DOM; the single render loop
  style.css                     the whole visual design system
  lyrics-data.mjs                real timing/content data (shared with tools/build-theatre-state.mjs)
  doodles/*.svg                  Lucide line-art icons, imported as raw strings and inlined
  scene/scene3d.js                renderer, camera, bokeh particles
  scene/photoLayers.js            Three.js sprite creation for the real photo cutouts
public/
  assets/theatre-state.json      generated Theatre.js sequence (do not hand-edit — regenerate instead)
  assets/photo/                  licensed, background-removed photo cutouts (see ATTRIBUTION.md)
  audio/                         the mp3 + its own README
tools/
  build-theatre-state.mjs        regenerates public/assets/theatre-state.json from src/lyrics-data.mjs
  smoke-test.mjs                 loads the generated state into real @theatre/core and samples a few values
  smoke-test-page.mjs            drives the real page (via `npm run dev`/`preview`) in headless Chromium
```

## Run locally

```bash
npm install
npm run dev       # Vite dev server with HMR
```

Or to check the exact production build:

```bash
npm run build
npm run preview   # serves dist/ at http://localhost:8080
```

Vite's dev/preview servers both honor HTTP Range requests, which audio
seeking depends on.

## Changing the choreography, timing, or photo layers

1. Edit `src/lyrics-data.mjs` (timing, words, section labels, which photo
   layer or doodle goes where — see `PHOTO_LAYERS`, `DOODLE_LAYERS`,
   `DEPTH_TIERS`, `MOODS`).
2. Edit the motion recipes in `tools/build-theatre-state.mjs` if you want to
   change the per-line variants, the camera, or how photo layers drift.
   Drop a new cutout in `public/assets/photo/` and reference it from
   `PHOTO_LAYERS` to add a new moment.
3. Run `npm run build:theatre` to regenerate `public/assets/theatre-state.json`.
4. `npm run smoke` samples a few values from the regenerated state (line
   motion, camera, a photo layer, mood) to sanity check it before you
   reload the page. `npm run smoke:page` drives the real page end to end in
   headless Chromium and screenshots a few timestamps — point `SMOKE_URL`
   at your dev server (and set `SMOKE_MOBILE=1` for a phone-sized viewport).

## Deploy

**Vercel:** import this GitHub repo at vercel.com/new. It auto-detects Vite
(`npm run build`, output `dist/`) — zero config.

**GitHub Pages:** build locally (`npm run build`) and deploy the `dist/`
folder, or wire up a GitHub Actions workflow that runs `npm run build` and
publishes `dist/`.
