# My Sweetheart

A single-page lyric experience for a genuinely public-domain 1910 love song
— built as a sequence of **vintage sunset posters**, matched frame-for-frame
to a Canva animation the sender designed as the reference. Each song section
is a fixed poster composition that builds in once and holds: a pinned
caption, fat retro display type stacking on word-by-word exactly as the
words are sung, one large photographic anchor object (a gramophone, a
rotary phone, a pocket-watch-era candlestick…) rising into place, the key
word writing itself in connected script across the lower third, thin line
doodles — on a brick-red → gold gradient. There is exactly one control: a
play button.

## What's actually in this build

- **Real audio, real timing, down to the word.**
  `public/audio/let-me-call-you-sweetheart.mp3` is a 1924 Jubileers
  vocal-quartet recording of the fully public-domain 1910 song "Let Me Call
  You Sweetheart" (Leo Friedman / Beth Slater Whitson). Every lyric and
  every word in `src/lyrics-data.mjs`'s `LYRICS` array carries its own
  start/end timestamp, produced by running
  [faster-whisper](https://github.com/SYSTRAN/faster-whisper) with
  word-level timestamps directly on that recording. See `ATTRIBUTION.md`.
- **Poster scenes, not a lyric scroller.** `SCENES` in `src/lyrics-data.mjs`
  divides the song at its section boundaries. Words accumulate onto each
  poster typewriter-style — every letter appears at a time derived from its
  word's real sung timestamps — and nothing ever flies through the frame.
  The word "sweetheart" (or the scene's key word) renders in connected
  script and reveals with a left-to-right ink wipe across its actual sung
  window.
- **Every tweened value is a real Theatre.js sequence.** Scene crossfades,
  each anchor's rise-in and idle float, doodle pops and wiggles, the
  gradient's hue/warmth shift between scenes — all keyframed tracks in
  `public/assets/theatre-state.json`, generated by
  `tools/build-theatre-state.mjs` (see that file for why it's generated
  rather than hand-authored in Theatre.js Studio) and evaluated by
  `@theatre/core` scrubbed to `audio.currentTime`. The letter reveals stay
  deliberately outside Theatre.js: a letter having appeared is discrete
  state at a timestamp, not an animated value.
- **Real photographic cutouts.** Every anchor is a licensed Adobe Stock
  photograph with its background removed (Photoshop API) — a genuine
  gramophone, rotary phone, dove, candlestick, pearl necklace, typewriter,
  bouquet, ring, vinyl disc, pocket watch. One per scene, chosen for the
  lyric it accompanies. See `ATTRIBUTION.md` for every source.
- **Depth without 3D.** Pointer-driven parallax (with a slow autonomous
  sway on touch devices) moves the anchor, doodles, and text at different
  rates through two CSS variables — the poster reads as layered depth with
  a 118 KB JS bundle and no WebGL.
- **Typography matched to the reference**: Titan One (fat retro display),
  Cookie (connected script), Oswald (caption) — self-hosted woff2, no
  external font dependency at runtime.
- **A real ending.** When the song finishes, the poster gives way to a
  closing card — scattered cutouts, "FROM NAVEEN, *with love*".
- **Mobile-first by construction**: the reference is a 9:16 Canva design,
  so the poster column *is* portrait — desktop centers the same column.
  Safe-area insets, `dvh` units, reduced-motion support throughout.

## Repo layout

```
index.html                     Vite entry (HTML shell only — see src/main.js)
src/
  main.js                       audio + Theatre.js wiring, letter reveal, parallax
  style.css                     the whole visual design system
  lyrics-data.mjs                word timing + SCENES poster definitions (single source of truth)
  doodles/*.svg                  Lucide line-art icons, inlined at build time
public/
  assets/theatre-state.json      generated Theatre.js sequence (regenerate, don't hand-edit)
  assets/photo/                  licensed, background-removed photo cutouts (see ATTRIBUTION.md)
  fonts/                         self-hosted Titan One / Cookie / Oswald (woff2)
  audio/                         the mp3 + its own README
tools/
  build-theatre-state.mjs        regenerates public/assets/theatre-state.json from src/lyrics-data.mjs
  smoke-test.mjs                 loads the generated state into real @theatre/core and samples values
  smoke-test-page.mjs            drives the real page in headless Chromium, screenshots each scene
```

## Run locally

```bash
npm install
npm run dev       # Vite dev server with HMR
```

Or the production build:

```bash
npm run build
npm run preview   # serves dist/ at http://localhost:8080
```

## Changing the posters, timing, or imagery

1. Edit `src/lyrics-data.mjs` — `SCENES` controls each poster's boundary,
   anchor photo/layout, gradient hue, and which word ranges land in the
   headline / script / sub / spoken slots.
2. Edit `tools/build-theatre-state.mjs` to change the build-in/float/fade
   choreography itself.
3. `npm run build:theatre` to regenerate the sequence, `npm run smoke` to
   sanity-check sampled values, and `npm run smoke:page` (with `npm run
   dev` running; `SMOKE_MOBILE=1` for a phone viewport) to screenshot every
   scene.

## Deploy

**Vercel:** auto-detected Vite (`npm run build`, output `dist/`) — zero
config. **GitHub Pages:** publish `dist/`.
