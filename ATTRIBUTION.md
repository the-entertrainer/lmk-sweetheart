## Lyric timing

Every timestamp in the `LYRICS` array in `src/lyrics-data.mjs` — both each
line's `start`/`end` and every individual word's `s`/`e` inside its `words`
array — was produced by running
[faster-whisper](https://github.com/SYSTRAN/faster-whisper) with word-level
timestamps directly on the recording above:

```python
from faster_whisper import WhisperModel
model = WhisperModel("small", device="cpu", compute_type="int8")
segments, info = model.transcribe(
    "let-me-call-you-sweetheart.mp3", word_timestamps=True, language="en"
)
for seg in segments:
    for w in seg.words:
        print(w.start, w.end, w.word)
```

then checked against the printed 1910 lyric sheet for spelling (Whisper's
transcription is used for **timing**, not wording). One exception: the final
tag line (the last ~1.4s of the recording, a soft fade-out) came back from
Whisper with all ten words collapsed onto the same timestamp — those ten
words were evenly re-interpolated across the segment's real start/end
instead of using the (degenerate) per-word output; every other line's word
timings are Whisper's actual output, unmodified.

## Motion / choreography

All motion in the current build is hand-authored CSS (transitions and a
few keyframe animations in `src/style.css`) driven by one
`requestAnimationFrame` loop in `src/main.js` that compares
`audio.currentTime` against the word timestamps above and toggles classes.
Word reveals, the keyword's ink wipe, the swash underline's
`stroke-dashoffset` draw, plate turns, and the colophon are all discrete
state changes at real timestamps — no animation library is used or
needed. (Earlier iterations drove tweened values through Theatre.js
keyframes generated by a build script; that machinery was removed with
the ink-editorial redesign. See git history if you need the
`__UNSTABLE_Project_OnDiskState` authoring example it contained.)

## Fonts

- **Instrument Serif** (Rodrigo Fuenzalida, Jordan Egstad) — the display
  face: roman for headlines, italic for the fountain-pen-blue "written"
  voice (keywords, the cover title, the colophon). The keyword reveals
  via a clip-path ink wipe rather than per-letter spans so the italic
  reads as one written word.
- **Schibsted Grotesk** (Schibsted, Jonathan Ring) — labels: the
  masthead, section names, small-caps credits. Variable `wght` axis,
  one file.
- **Fragment Mono** (Wei Huang) — the apparatus: plate numbers,
  timecode, folios, figure captions, spoken banter lines, the colophon's
  printer's notes.

All three SIL Open Font License, self-hosted as latin-subset woff2 files in
`public/fonts/` (downloaded from Google Fonts) — no external font request
at runtime. (The previous build's Titan One / Cookie / Oswald files were
removed with the redesign.)

## Design model

The current build — "The Sweetheart Edition" — is an ink-on-paper
editorial direction: the song typeset live as a fine-press love letter
(paper ground, hairline trim, plates, folios, colophon), with cinematic
masked type reveals in the lineage of film title sequences. The plate
structure itself (fixed composition per song section, word-by-word
accumulation at sung timestamps, one photographic object per section, an
emphasized keyword, the sender's caption line kept verbatim) carries over
from the sender's original Canva reference animation.

## What changed from the previous builds

The prior iteration of this page used three procedurally-authored Three.js
glass objects (a heart, an abstracted bloom, a ribbon) and a curated set of
flat-color OpenMoji PNG icons on a plum/blush palette. That direction was
fully replaced after the user shared a Canva animation showing their actual
intended look: a warm vintage sunset palette, real photographic vintage
objects instead of abstract glass or flat icons, and a rounded-display +
script-accent type pairing. The glass-object and OpenMoji-icon assets from
that build were removed rather than kept alongside the new direction, to
avoid two competing visual languages on the same page.

## What changed again: the TV-broadcast pivot

The DOM/CSS poster-scene build above was itself later replaced: the whole
experience played out as a broadcast on a centered 3D vintage TV's CRT
screen, with the typography and photographic anchors drawn onto a canvas
composited into the model's screen texture and vintage
scanline/static/glitch effects layered on top. Three.js was reintroduced
for that pivot.

## What changed last: the ink-editorial redesign ("The Sweetheart Edition")

The TV build was in turn fully replaced by the current direction on an
explicit request to reimagine everything — approach, palette, fonts,
graphics, theme, animations. The 3D TV model and Three.js went away
(nothing WebGL remains), the vintage-warm palette became ink on paper
(bone paper, letterpress ink, fountain-pen blue, one seal red), Titan
One/Cookie/Oswald became Instrument Serif/Schibsted Grotesk/Fragment
Mono, and the CRT broadcast became a fine-press edition of ten plates.
The word-level timing data, the licensed photo cutouts (now printed as
grayscale ink plates), the recording, and the one-button interaction are
the only things that carried over.
